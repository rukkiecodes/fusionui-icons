# FusionUI Icons

A searchable gallery for **FusionUI Icons** — a clean, consistent stroke-icon set
for the [FusionUI](https://github.com/rukkiecodes) design system that unifies UI
across web and mobile.

🔗 **Live site:** https://rukkiecodes.github.io/fusionui-icons/

![icons](https://img.shields.io/badge/icons-737-195bff)

## Features

- 🔍 **Instant search** across icon names and semantic keywords (e.g. `close`, `cart`, `gear`).
- 🧭 **Paginated grid** — 200 icons per page for fast rendering.
- 🎨 **Theme aware** — follows your **system** theme by default; switch to Light or Dark anytime.
- 📋 **One-click copy** in four formats:
  - **SVG (HTML)** — paste straight into any page
  - **Vue** — `<FIcon :icon="..." />` with `@rukkiecodes/icons`
  - **JSX / React** — a ready-to-use component
  - **Name** — the kebab-case icon id

## Usage

Click any icon to open it, then copy the snippet you need. Press <kbd>/</kbd> to
jump to the search box and <kbd>Esc</kbd> to close the dialog.

```html
<!-- Example: copied "SVG (HTML)" output -->
<svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24"
     fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round"
     stroke-linejoin="round" class="fui fui-activity">
  <polyline points="22 12 18 12 15 21 9 3 6 12 2 12"></polyline>
</svg>
```

Icons inherit the surrounding text `color` (via `currentColor`) and scale with
`width`/`height` or `font-size`.

## Development

```bash
npm install      # install deps
npm run dev      # start the dev server
npm run build    # production build to dist/
npm run preview  # preview the production build
npm run icons    # regenerate src/icons.json (+ svg/) from the icon source
```

### Icon data

The gallery is driven by `src/icons.json`, generated by
`scripts/generate-icons.mjs`. The script reads the SVG source (the committed
`svg/` folder, or the FusionUI monorepo if present), rebrands the markup to the
`fui-*` class convention, and emits a compact dataset. Re-run `npm run icons`
whenever the icon set changes.

## Deployment

Pushing to `main` triggers the GitHub Actions workflow in
`.github/workflows/deploy.yml`, which builds the site and publishes it to
GitHub Pages. The Vite `base` is set to `/fusionui-icons/` to match the project
Pages URL.
